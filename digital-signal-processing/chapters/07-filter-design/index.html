    Theory

    All classical IIR design methods start from an analog prototype and convert to digital via the bilinear transform. The main prototypes differ in how they distribute approximation error:

    PrototypePassbandStopbandTransitionOrder for spec
    ButterworthMax. flatMonotonicWidestHighest
    Chebyshev IEquirippleMonotonicMediumMedium
    Chebyshev IIFlatEquirippleMediumMedium
    EllipticEquirippleEquirippleNarrowestLowest

    The Butterworth prototype is the simplest and has maximally flat magnitude:

    $$|H_a(j\Omega)|^2=\frac{1}{1+(\Omega/\Omega_c)^{2N}}$$

    The Chebyshev Type I trades passband flatness for a sharper transition:

    $$|H_a(j\Omega)|^2=\frac{1}{1+\epsilon^2 T_N^2(\Omega/\Omega_c)}$$

    where $T_N$ is the Chebyshev polynomial of degree $N$.

    The Elliptic (Cauer) filter achieves the absolute minimum order for given passband ripple $R_p$, stopband attenuation $A_s$, and transition ratio — at the cost of ripple in both bands.

    A complete filter design flow involves these steps:

    1. Specification: Define passband edge $\omega_p$, stopband edge $\omega_s$, passband ripple $R_p$ dB, and stopband attenuation $A_s$ dB.
    2. Choose topology: FIR (if linear phase required) or IIR (if efficiency is priority).
    3. For IIR:
      1. Pre-warp: $\Omega_p = \frac{2}{T}\tan(\omega_p/2)$, $\Omega_s = \frac{2}{T}\tan(\omega_s/2)$.
      2. Choose prototype (Butterworth/Chebyshev/Elliptic).
      3. Determine minimum order from selectivity ratio and attenuation spec.
      4. Design analog prototype $H_a(s)$.
      5. Apply bilinear transform to get $H(z)$.
    4. For FIR:
      1. Estimate order from Kaiser formula: $N\approx\frac{A_s-7.95}{2.285\,\Delta\omega}+1$.
      2. Design windowed-sinc or use Parks-McClellan algorithm.
    5. Verify: Check magnitude response, phase/group delay, and stability.
    6. Implement: Realize as cascade biquad sections (IIR) or direct convolution (FIR).

    Multirate DSP processes signals at different sampling rates within the same system.

    Decimation (Downsampling by M)

    Keep every $M$-th sample: $y[n]=x[nM]$. The spectrum of $y$ contains $M$ shifted, scaled copies of $X(e^{j\omega})$:

    $$Y(e^{j\omega})=\frac{1}{M}\sum_{k=0}^{M-1}X\!\left(e^{j(\omega-2\pi k)/M}\right)$$

    An anti-aliasing filter (lowpass with cutoff $\pi/M$) must precede the downsampler.

    Interpolation (Upsampling by L)

    Insert $L-1$ zeros between each sample, then apply an anti-imaging filter (lowpass with cutoff $\pi/L$, gain $L$):

    $$x_u[n]=\begin{cases}x[n/L],&n=0,\pm L,\pm 2L,\dots\\0,&\text{otherwise}\end{cases}$$

    Noble Identities

    Filters can be interchanged with up/downsamplers under polyphase decomposition, enabling efficient implementations.

    Coefficient Quantization

    In fixed-point implementations, filter coefficients must be quantized. This shifts poles and zeros, potentially changing the response significantly. Cascade second-order sections minimize this sensitivity.

    Overflow and Scaling

    Fixed-point arithmetic can overflow. Proper scaling at each biquad stage ensures the signal stays within the representable range.

    Round-Off Noise

    Each multiply-accumulate introduces quantization noise. For an $N$-th order direct-form IIR, the output noise variance grows with $N$. Cascade and parallel forms reduce this.

    Computational Complexity

    StructureMultiplies/sampleAdds/sample
    FIR Direct Form (order $N$)$N+1$$N$
    IIR Direct Form II (Nb, Na)$N_b+N_a+1$$N_b+N_a$
    Cascade Biquads (K sections)$5K$$4K$

    Real-time filtering processes data in blocks (frames) or sample-by-sample:

    Block Processing

    Collect $N$ input samples, apply the filter (possibly via FFT-based fast convolution for long FIR), output $N$ filtered samples. Introduces latency of $N$ samples.

    Overlap-Save / Overlap-Add

    For FIR filters of length $M$ and block size $N$:

    • Overlap-Save: Append new $N$ samples to previous $M-1$, FFT the block of $N+M-1$, multiply with $H[k]$, IFFT, keep last $N$ samples.
    • Overlap-Add: Zero-pad each block to $N+M-1$, FFT-convolve, overlap-add adjacent results.

    Both achieve $O(N\log N)$ per block instead of $O(NM)$.

    Sample-by-Sample

    Use the difference equation directly with circular buffers for delay lines. Minimal latency (1 sample) but no FFT speedup.

    Key Equations

    Butterworth Order

    $$N\ge\frac{\log\!\left[\frac{10^{A_s/10}-1}{10^{R_p/10}-1}\right]}{2\log(\Omega_s/\Omega_p)}$$

    Minimum order for Butterworth filter given passband ripple $R_p$ and stopband attenuation $A_s$.

    Decimation Spectrum

    $$Y(e^{j\omega})=\frac{1}{M}\sum_{k=0}^{M-1}X(e^{j(\omega-2\pi k)/M})$$

    Downsampling creates $M$ spectral copies. Without anti-aliasing, these overlap causing aliasing distortion.

    Overlap-Save

    $$y[n]=\text{IFFT}\{X[k]\cdot H[k]\}$$

    Fast convolution via FFT: compute circular convolution, discard first $M-1$ samples per block.

    Cascade Biquad

    $$H(z)=\prod_k\frac{b_{0k}+b_{1k}z^{-1}+b_{2k}z^{-2}}{1+a_{1k}z^{-1}+a_{2k}z^{-2}}$$

    Preferred IIR implementation. Minimizes coefficient sensitivity and finite-precision effects.
